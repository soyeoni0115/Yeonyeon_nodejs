## test 패키지 = jest
- 테스트를 아무것도 작성하지 않으면 에러 발생 = 테스트 실패
- test함수의 첫 인수로 테스트에 대한 설명 작성, 두번 쨰 인수엔 테스트 내용 작성
```javascript
test('1+1은 2입니다.',() =>{
    expect(1+1).toEqual(2);
});
```
- expect에 넣은 값과 toEqual에 넣은 값이 일치하면 테스트 통과

## 모킹
- 가짜 객체, 가짜 함수를 넣는 행위

## 유닛테스트
- 작은 단위의 함수나 모듈이 의도된 대로 정확히 작동하는지 테스트하는 것
- 나중에 함수를 수정하면 기존에 작성해둔 테스트가 실패되기에 함수 수정 시 어느 부분이 고장 나는지를 테스트를 통해 알 수 있음

## 테스트 커버리지
- 테스트 결과가 출력되고 표가 출력 됨. 해당 표에서 퍼센티지가 높을수록 많은 코드가 테스트 된 것

## 통합 테스트
- 하나의 라우터를 통째로 테스트하여 이런 것들이 모두 유기적으로 잘 작동하는지 테스트하는 것

## 부하 테스트
- 서버가 얼마만큼의 요청을 견딜 수 있는 지 테스트 하는 것
- 내 코드가 실제로 배포되었을 때 어떤 문법적, 논리적 문제가 있을지 유닛 테스트와 통합테스트를 통해 파악 가능하나, 서버가 몇 명의 동시 접속자나 일일 사용자를 수용할 수 있는지 예측하는 건 더 어려움
